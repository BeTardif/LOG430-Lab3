### Definitions et principes ###
* **API REST**: Un API REST est une interface qui permet la communication d'application via le web en utilisant des methodes HTTP. Rest est un acronyme pour *Representational State Transfer*. 
* **ER (Entity relationship)**: Modèle conceptuel des données. Permet la visulatisation des relations entre les données.
* **GRAPHQL**: Language deprogrammation peremettant des requêtes de données basées sur des schémas de structures de données.
* **FLASK**: Framework Python pour déveloper des API web
* **MYSQL Workench**: Outil de visualisation de base données. Il permet certaines interactions comme la création de tables et de relations.
* **Principes Restful**: Les principes Restful sont basés des échanges de requêtes et de réponses des standards HTTP. Ces méthodes de bases sont les suivantes:  
    GET: Permet d'obtenir des données;  
    POST: Créer un nouvel objet avec des données;  
    PUT: Actualise des données d'un objet;  
    DELETE: Retire des données.
